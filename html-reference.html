HTML <html>and </html> HTML is the main coding for websites on the internet. It represents Hypertext Markup Language which the internet can run on. This language is used all throughout the code. At the beginning of any code that is HTML, there must be a "!DOCTYPE html" tag and a "/html" tag insert link here
Headers <head>and </head> A header is the title and styles of a page go, it will represent what the website is about and how it looks. Different styles can be used to format the page using CSS. When putting multiple headers, there must be a difference or else the heads would be the same. Use "h #" tag when using multiple headers.
Title <title> and </title> The title is a format that makes words different to the normal text. This does not necessarily need to be in the header. It can be used when marking everying. When using different formats for different titles, there must be a different tag. "t #" is one way to clarify.(# means title number)
Body <body>and </body> The body is where the main code and information goes. These are made to separate the head and the information that is desired. Usually, the tag "body" to start the body and end it with "/html".
Headings <h>and </h> When writing headings, it is important to differenciate it from other text. If not, the reader may become confused. Using the "h" tag, headers are applied. Adding a number ahead of it will assist in classification.
Paragraphs <p>and </p> When writing paragraphs, the tag "p" is used. It is used when writing large amounts of information that is supposed to show up on the page.
Breaks <br>and </br> When finding ways of organization, breaks in the page may help. By using the tag, the content would separate.
Lines <hr>and </hr> When requiring more obvious organization, coding lines may not be a bad idea. They can help in separating content.
Comments "<(!----)>and </(!----)>" When writing code, comments are very important. It is to remind the coder what improvements may be needed. Comments Exercises
Links "<a href> and </a href>" When writing content, links to other websites may not be a bad idea. It assists in stating sources and helping others find information. Links Exercises
Images <img>and </img> When writing content, images can help in explanations or give a visual understanding. However, it references it from a webpage and the name must be accurate. Images Exercises
Table <table>and </table> <tr> and </tr><th> <td>and </td> When displaying data, table may come in handy. They represent data. This reference page is using a table. Tables Exercises
Blocks and <div>and </div> When displaying data, blocks are good because they create a very effective means of separating data into compartments. Blocks examples

Table element

Description

It creates a table

Syntax

Start tag: <table>

Content: text

End tag: </table>

Example

<table style="width:100%">

<tr>

<th>Firstname</th>

<th>Lastname</th>

<th>Age</th>

</tr>

</table>

Display

Firstname Lastname Age
